Plugins

Cliniface's functionality can be extended by third-party developers via the development and distribution of plugins which take the form of separately compiled shared libraries (known as Dynamically Loaded Libraries or DLLs in Windows). Some plugins are included in the base Cliniface platform, but others can be obtained directly from the plugin developers.

It cannot be guaranteed that plugins developed by third-parties will work correctly in Cliniface or that they will not cause errant behaviour in Cliniface. If you require assistance in the use of a particular Cliniface plugin, you should contact the provider/developer of that plugin for support.

Loading Plugins

Upon starting, Cliniface looks for plugins within its plugins directory which is located within the bin directory of Cliniface's installation directory. If plugins are added to this directory after Cliniface starts, you must restart Cliniface to reparse the plugins directory and attempt to load the newly added plugin(s). Plugins that are successfully loaded are shown in the plugins dialog (accessible from the Help menu) in black text. Plugins that did not load successfully are shown in red. If a plugin doesn't load, this can be for several reasons, but the most common reason is that the plugin does not provide Cliniface with the correct compliance code upon loading. Compliance codes for plugins may be updated in newer versions of Cliniface and so plugins developed for older versions of Cliniface may not work with more recent versions of Cliniface. This is intentional since this provides Cliniface with a "scorched earth" mechanism to prevent third-party plugins from being used in later versions should any undesirable issues arise with them. If you currently use a third-party plugin for an older version of Cliniface which doesn't work in a newer version of Cliniface, please contact the provider of that plugin for an updated version.
